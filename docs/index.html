Features

Built for the shape of the problem, not a generic dashboard.

Every session's PreToolUse hook pipes here. You approve from one place, with risk already classified.

Pending at a glance

Menu bar icon has three states: idle, waiting, auto-allow active. State is carried by shape, not color — macOS template compliant.

Batch or granular

Allow / Deny individually, Allow-for 2 / 10 / 30 min for temporary trust windows, or ⌘↩ to clear the queue.

Risk hints built in

rm -rf, sudo, curl | sh, writes to /etc /usr /System flagged red; any Edit / Write / WebFetch gets an amber edge.

Pin to stay open

Popover auto-closes on outside click by default. Pin it and it survives app switches, Space changes, and full-screen toggles — review while you keep working.

Native & local-only

Embedded HTTP + WebSocket server on 127.0.0.1:7788. No Python, no Node. Speaks Claude Code's official PreToolUse hook.

Sparkle auto-update

EdDSA-signed incremental updates via Sparkle. Silent background checks; you approve the install.

How it works

Five hops, all local, never blocking.

  1. 01
    Claude Code CLI
    Your session is about to run a tool.
  2. 02
    PreToolUse hook
    Official Claude Code hook fires with tool name + input.
  3. 03
    127.0.0.1:7788
    Local HTTP/WS server receives the request. Nothing leaves your machine.
  4. 04
    Popover
    Menu bar shows the approval card. You click Allow / Deny / Allow for 10 min.
  5. 05
    CLI continues
    Decision returns; the tool runs or is blocked.
Never blocks the CLI

If the app isn't running — or the health check times out after 2s — the hook silently falls back to Claude Code's native TUI prompt. Quit the app mid-session and your terminal still works.

FAQ

Questions the first 50 installers asked.

Why isn't this on the Mac App Store?
The sandbox rules can't accommodate reading ~/.claude/settings.json and a local HTTP server from a user-facing menu-bar surface. We went the Developer ID + Apple-notarized route instead — same trust level, wider capabilities.
Does it modify my ~/.claude/settings.json?
Yes. Five PreToolUse hook entries get appended on first launch. The original file is backed up to ~/.claude/settings.backup-{timestamp}.json. Uninstall removes both.
What happens when the app isn't running?
The hook performs a 2-second health check against 127.0.0.1:7788/healthz. If it fails, control is handed back to Claude Code and you get the native TUI prompt. Zero blocking, even if the app crashes.
Is it open source?
MIT, github.com/heypandax/cc-dashboard. PRs welcome — see CONTRIBUTING.md for the build prerequisites.
What data gets collected?
Anonymous install + launch counts via Firebase, and crash reports via Crashlytics. Command content, file paths, and cwd values never leave your Mac. One command to opt out entirely: 
defaults write com.heypanda.cc-dashboard analyticsEnabled 0
Why port 7788?
Unregistered in IANA, rarely clashes with dev tooling. Configurable via defaults write com.heypanda.cc-dashboard serverPort 9988.
